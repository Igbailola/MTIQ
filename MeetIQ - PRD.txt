1. Product Overview
MeetIQ is an AI-powered execution and accountability platform that transforms meeting conversations into confirmed commitments, ownership, and measurable outcomes.

Instead of simply generating meeting notes, MeetIQ ensures that decisions made during discussions become actionable, assigned, acknowledged, and tracked until completion.

MeetIQ serves as the missing layer between conversation and execution, helping teams improve accountability, visibility, and follow-through.

2. Problem Statement
Teams spend countless hours in meetings discussing priorities, making decisions, and assigning responsibilities.

However, after meetings end:

Decisions are forgotten

Ownership becomes unclear

Action items are buried in notes

Deadlines are missed

Accountability becomes difficult to track

Most existing tools focus on documenting meetings rather than ensuring execution.

The result is slower project delivery, reduced productivity, and poor operational visibility.

3. Solution
MeetIQ converts meetings into structured commitments that can be tracked from creation to completion.

Using AI, MeetIQ analyzes uploaded meeting notes or transcripts and automatically:

Generates meeting summaries

Extracts decisions

Identifies action items and converts them into commitments

Suggests owners based on who said what in the transcript

Detects deadlines

Flags ambiguous responsibilities

MeetIQ then transforms these outputs into confirmed commitments that team members can acknowledge, manage, and complete.

This creates a clear execution workflow that ensures discussions lead to measurable outcomes.

4. Product Vision

MeetIQ aims to help organizations ensure that every meeting produces ownership, accountability, and progress.

10.4 AI Processing Engine
Generate:

Meeting Summary (structured, 3-5 bullet sections)

Key Decisions (with context snippets)

Action Items → Commitments

Suggested Owners (with confidence score)

Suggested Deadlines (with confidence score)

Ambiguity Detection (flagged items)

Speaker-Aware Extraction:

The AI prompt is explicitly instructed to:

Identify speakers by name wherever possible (e.g., "Tolu said…", "Sarah will…").

Extract commitments directly from the speaker's own words or from explicit delegation.

When a speaker claims a task ("I’ll design the landing page"), assign that speaker as the owner.

When a task is delegated ("Tolu, can you handle the design?"), assign the named person as the owner.

Map extracted names to workspace members when suggesting owners. If no match exists, leave the suggestion empty and flag as "Needs Owner".

For every commitment, include a short quote or line reference from the transcript that supports the extraction.

Confidence Scoring:

Each AI output displays a confidence indicator:

High Confidence (Green): Clear owner and deadline detected

Medium Confidence (Yellow): Some interpretation required, review recommended

Low Confidence (Red): AI uncertain, manual review required

AI Disclaimer:

Every AI-generated section carries a subtle "AI Generated" label with a tooltip: "This output was generated by AI. Please review for accuracy before publishing."

Processing Time:

Target: <30 seconds for standard meeting notes

Loading state: Skeleton cards with pulsing animation and "Analyzing your meeting..." message

AI Feedback Loop (Thumbs Up/Down):

Each AI output includes 👍 👎 buttons

Thumbs down opens a quick feedback form: "What was incorrect? (Owner / Deadline / Action Item / Summary)"

Feedback is stored and used to fine-tune future model behavior

Aggregated feedback is reviewed weekly by the product team

10.6 Commitment Confirmation System
Assigned users must explicitly respond to each commitment. This ensures accountability is explicit rather than assumed.

Confirmation Options:

Accept Commitment (green confirmation)

Reject Commitment (reason required)

Request Changes (specify what needs updating)

Confirmation Flow Details:

AI generates commitments post-upload

Workspace admin/uploader reviews and publishes

Assigned owners receive in-app notification

Owners have 48 hours to respond

Unconfirmed commitments are flagged and highlighted on dashboard

After 48 hours, workspace admins receive escalation notification

Rejected or change-requested items return to uploader for reassignment

Uploader can reassign, edit, or dismiss

Escalation Path:

24h after assignment: Gentle reminder notification

48h after assignment: Escalation to workspace admin

72h after assignment: Commitment auto-marked as "At Risk"

Micro-interactions:

Accept: Green checkmark animation (200ms ease-out)

Reject: Red pulse with reason modal

Request Changes: Yellow highlight with inline comment field

11. User Stories (Enhanced with Acceptance Criteria)
Meeting Processing

As a user, I want to upload meeting notes so AI can analyze them and generate structured outputs.

Acceptance Criteria:

User can upload .txt, .docx, or .pdf files

User can paste raw text directly into a text area

Upload triggers AI processing within 30 seconds

User sees loading state during processing

User sees error message if upload fails or format is unsupported

User is notified when processing is complete

Summaries

As a user, I want AI-generated summaries so I can quickly understand meeting outcomes.

Acceptance Criteria:

Summary appears in a visually distinct "AI Generated" card

Summary is 3-5 bullet points max

Summary includes key topics and conclusions

User can edit the summary after generation

Summary displays confidence indicator

Commitment Extraction

As a user, I want AI to identify action items and convert them into commitments.

Acceptance Criteria:

Action items are extracted and displayed as discrete commitment cards

Each commitment includes suggested owner and deadline (if detected)

Commitments without detected owners are flagged "Needs Owner"

User can add, edit, or delete commitments before publishing

AI confidence is displayed on each commitment

Ownership

As a user, I want responsibilities assigned so accountability is clear.

Acceptance Criteria:

AI suggests owners based on meeting context

User can reassign owners via dropdown

Owner receives in-app notification upon assignment

Dashboard shows per-owner commitment breakdown

Confirmation

As an assignee, I want to confirm ownership so there is no ambiguity around responsibilities.

Acceptance Criteria:

Assignee sees "Pending Confirmation" items in their dashboard

Assignee can Accept, Reject, or Request Changes

Accept triggers green confirmation animation

Reject requires a reason (free text, min 10 characters)

Request Changes requires a note explaining what needs updating

Confirmation status updates in real-time across workspace

Tracking

As a user, I want to monitor commitment progress from a central dashboard.

Acceptance Criteria:

Dashboard shows all commitment statuses

Overdue items are visually distinct (red highlight)

User can filter by status, owner, or meeting

KPI cards update in real-time

User can click into any commitment for full detail

15. Data Privacy, Retention & Security
Data Handling:

Uploaded meeting content is encrypted at rest (AES-256) and in transit (TLS 1.3)

AI processing occurs via secure API (OpenAI) with data not used for model training

Users can delete their data at any time (workspace deletion cascades)

Meeting content is retained for the life of the workspace or until manual deletion

Compliance (Out of Scope for MVP, on Roadmap):

SOC 2 Type II (Phase 3)

GDPR compliance baseline in MVP (data export, deletion on request)

SAML/SSO for enterprise (Phase 3)

MVP Security Commitments:

Supabase Row Level Security on all tables

Rate limiting on API routes

Input sanitization on all user-provided content

No meeting data used for AI model training

16. User Onboarding Flow
First-Time User Experience:

User signs up with email + password (or Google OAuth).

Email verification (required before first upload).

After verification, a 3-step onboarding wizard is shown (can be skipped):

Step 1: Welcome – set display name and timezone.

Step 2: Create your workspace (or join an existing one via invite).

Step 3: Upload your first meeting transcript or try a demo transcript to see MeetIQ in action.

User lands on the dashboard with a guided tooltip walkthrough (key elements highlighted).

Onboarding is considered complete when the user uploads their first meeting and confirms at least one commitment.

Onboarding success measured by: % of new users who upload a meeting within 48 hours of signup.
